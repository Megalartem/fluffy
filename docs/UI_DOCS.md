# UI Guide v1 — правила интерфейса

Этот документ фиксирует единые правила UI для приложения. Его цель — убрать вкусовщину, ускорить разработку и сохранить цельный характер интерфейса.

---

## 1. Цель UI

UI должен:

* ускорять **фиксацию трат и доходов**
* быть **дружелюбным, но не стимулирующим**
* поддерживать работу с целями (там допустим более приятный визуальный отклик)
* одинаково хорошо работать на **mobile и desktop**

**Формула:**

> быстро записать → спокойно посмотреть

---

## 2. Визуальная основа (Airbnb-inspired)

Мы **не копируем** Airbnb один-в-один, но берём их принципы: чистота, тёплые нейтрали, “air”, аккуратные радиусы и мягкая глубина.

### Тон

* спокойный, дружелюбный, без давления
* интерфейс «приглашает» к действию, но не стимулирует дофамином

### Контраст

* средний (читабельно, но не жёстко)
* основной контраст — типографика + spacing, а не цвет

### Цвет

* базовые поверхности — тёплые нейтрали
* акцент (brand) — **Airbnb coral** (CTA, активные состояния, focus)
* цели — могут использовать **зелёный** для прогресса (но спокойный)
* бюджеты — warning/danger без «праздников»

---

## 3. Layout и отступы

* mobile-first
* используем воздух вместо линий
* карточки предпочтительнее таблиц

**Правило:** если хочется добавить границу — сначала увеличь отступ.

Рекомендации:

* `p-4 / p-6`
* `space-y-3 / 4 / 6`

---

## 4. Поверхности и карточки

По умолчанию:

```
bg-surface border border-border rounded-xl
```

Ключевые элементы:

```
shadow-lift
```

Списки:

* без теней
* hover через `bg-surface-2`

---

## 5. Тени (мягкая глубина как у Airbnb)

Тени — не «эффект», а способ отделить слои.

Разрешено:

* FAB
* Modal / Sheet
* ключевые карточки Dashboard
* hover/focus у интерактивных элементов

Запрещено:

* тяжёлые тени на каждом элементе списка
* “материальные” резкие тени

Принцип: лучше **тонкая тень + border + фон**, чем одна сильная тень.

---

## 6. Скругления (Airbnb-like)

* базовые элементы: `rounded-xl`
* ключевые контейнеры: `rounded-2xl`
* избегаем излишне «пухлых» pill-форм по умолчанию

---

## 7. Типографика

* заголовки умеренные
* жирность используется точечно
* числа и суммы — всегда `tabular-nums`
* не используем pure black

---

## 8. Компоненты

### Button

* один `primary` на экран
* `secondary` — второстепенные действия
* `danger` — только для удаления

### Input

* ошибки inline под полем
* focus через ring
* placeholder — нейтральный

### Modal / Sheet

* overlay: `bg-black/30`
* panel: `bg-surface border-border shadow-float`
* ESC закрывает
* body scroll блокируется

---

## 9. Состояния

### Ошибки

* коротко
* рядом с местом проблемы
* без кодов ошибок

### Пустые состояния

* 1 фраза
* 1 действие

### Loading

* минимальный
* без лишних анимаций

---

## 10. Фильтры и сортировка

* фильтры спрятаны под иконкой
* активные фильтры имеют индикатор
* фильтры храним в state
* сортировка: desc → asc → default

---

## 11. Эмоции и «праздник»

Разрешено:

* мягкие reward-эффекты в целях

Запрещено:

* дофаминовая стимуляция в операционке
* конфетти, вспышки, яркие анимации

---

## 12. Definition of Done для UI

Перед merge:

1. Удобно одной рукой на mobile
2. Один главный CTA
3. Нет лишних теней
4. Используются только токены
5. Числа с `tabular-nums`
6. Ошибки inline
7. Нет оценочных формулировок

---

## 13. Структура

* tokens: `src/shared/design/tokens.css`
